# Hardening validation gates, schemas, and pipeline-trust primitives

A gate/schema/integrity check certifies only **what it can see**. Every recurring failure below is a way it silently passes a defect (false-GREEN / under-catch) or permanently blocks legitimate input (false-RED / over-catch). Front-load the relevant checklist; do not discover these one adversarial round at a time.

## Cross-cutting spine (applies to every gate here)

1. **Positive allowlist, fail closed — never a denylist.** Enumerating what to reject gets broken by a new shape every review round; a denylist that blocks `/ \ ..` still passes shell metacharacters. State the finite set of *accepted* shapes; everything else halts.
2. **Strictness bias — the only tolerable failure is a false-HALT, never a false-PASS.** For any ambiguous design choice pick the branch whose worst outcome is "safely refused" (remedy is idempotent) over "silently accepted a bad value." *When unsure, HALT.*
3. **Red-before-green is necessary but NOT sufficient — prove BOTH directions.** Watch the gate FAIL on the real defect, AND separately construct realistic *legitimate* content and prove it does NOT trip. A gate that over-catches hand-authored/LLM content is often worse than the under-catch it replaced.
4. **Front-load the whole checklist.** A single adversarial reviewer surfaces these one class per round, in the same order, even on a same-plugin precedent file. Paste the checklist into the plan up front instead of looping N times.
5. **Differential-test against a REAL oracle, then audit the harness.** Compare your reader against `ruby -ryaml` / `jsonschema` / `Psych` over hand-fixtures AND an adversarial (not ASCII-only) fuzz corpus. A green run over a blind corpus proves only the classes you fuzzed — run ≥1 adversarial reviewer as the backstop for the class you didn't think to fuzz. Also audit that the harness implements the spec (using `.trim()`/`\s` where the spec says ASCII-only *proves the wrong program*).
6. **Convergence is a SIMPLIFICATION, not another layer.** When a reviewer keeps returning same-CLASS collisions/false-passes, DELETE the clever machinery generating them (the sorter, the sentinel, the field projector) rather than adding context-awareness to it. A fix round that deletes code is converging; one that adds a guard is usually whack-a-mole.
7. **Schema is advisory unless `jsonschema.validate` actually runs at runtime — grep for it first.** If nothing validates the doc at runtime, the schema `pattern`/`enum` is documentation; the per-script runtime check is the load-bearing thing.
8. **Independent oracle — a gate that derives its expectation THROUGH the code under test is blind by construction.** If `expected` is computed by calling the same function whose output you are validating, `expected == actual` always holds and the gate exits 0 on every defect in that function — green forever, by design, not by bug. Ask of every gate: *where does its expectation come from, and could it disagree with the thing it checks?* Corollary: never cite such a gate as evidence a fix works. → [`references/gate-review-lenses.md`](references/gate-review-lenses.md)
9. **An empty-result sentinel that means "nothing to do" also means "nothing was detected".** A producer emitting `{no_new_candidates: true}` / an empty set / zero rows conflates *the work is already complete* with *my detector found nothing* — and consumers reliably assume the benign one, skipping the step that would have surfaced the failure. Make the two states distinguishable at the source, or have the consumer prove completeness independently. → [`references/gate-review-lenses.md`](references/gate-review-lenses.md)
10. **Never let the value under test help DERIVE the reference it is tested against.** A containment gate ("X must resolve under root R") is silently destroyed if R is derived by searching for a root that *contains* X: the gate becomes vacuous at best, and at worst the search widens R until X fits — so the exact violation the gate exists to catch now PASSES (false-GREEN), and the wrong-but-wider R is then used for real output. Derive R from one independent anchor, test every other path against it, and on multiple candidate roots **HALT as ambiguous** rather than picking innermost/outermost — a tie-break here is a silent wrong answer. This is the derivation-side twin of lens 8 (independent oracle): 8 is about where an *expectation* comes from, this is about where a *reference frame* comes from. → [`references/gate-review-lenses.md`](references/gate-review-lenses.md)
11. **A deferred/found instance signals a CLASS.** State the invariant that bounds the whole class and open ONE tracking issue; do not patch instance-by-instance. Resolving error A by moving the same input's failure to error B is zero progress — enumerate the full failure set for the topology the fix targets.
12. **A green assertion proves ONE rung of a ladder, not the claim you read into it.** For prose/doc-contract gates (fixed-string, section-scoped, count- or line-order-based) the rungs are distinct and independent: *exists anywhere* → *in the right section* → *is NEW rather than a pre-existing attractor* → *in the right PLACE within that section* → *anchored to something unrenameable*. Passing rung N says nothing about N+1. Two orthogonal checks belong with it: **satisfiability** — individually-reasonable assertions can be JOINTLY impossible (a required sentence whose own identifier a negative pin forbids in the same section), so review PAIRS, never single assertions; and **witness completeness** — a requirement with N normative outcomes needs N witnesses, or the implementer writes the setup plus one and omits the rest with everything green. The ladder never terminates in a fixed-string world, so bound the threat model in the artifact (careless implementation and drift, *not* a hostile author) and file the structure-aware parser. → [`references/gate-review-lenses.md`](references/gate-review-lenses.md) §7
13. **A destructive filesystem gate — "clean the managed dir, then rebuild" — is a symlink DATA-LOSS / out-of-root-write surface.** Every dotfile you PRESERVE across the clean (an ownership marker, its write-temp, snapshot dirs) is a symlink-survival vector, and `is_file()`/`is_dir()`/`mkdir(exist_ok=True)`/`Path.resolve()` ALL follow symlinks — so a planted symlink turns the clean into an `rm` into someone else's tree and a write into an out-of-root clobber, invisibly (a green suite hides every vector). Require REAL regular files (`is_symlink()` BEFORE `is_file()`), write atomically via `mkstemp`(`O_EXCL`, no-follow)+`os.replace`, normalize paths NO-FOLLOW (`os.path.abspath` + reject `..`, NEVER `.resolve()` upstream of the guards), and stop the symlink walk at the trusted install root — with a false-RED tail: don't over-reject real OS symlinks (`/var`,`/tmp`) on an out-of-root path. → [`references/gate-review-lenses.md`](references/gate-review-lenses.md) §9
